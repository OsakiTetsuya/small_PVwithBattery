# CLAUDE.md - 開発メモ

## プロジェクト概要

低圧太陽光発電所とDCリンク蓄電池を組み合わせた場合のJEPX売上シミュレーター（MVP版）

## 実装状況

### 完了した機能

1. **プロジェクト構造**
   - ディレクトリ構成を仕様書通りに作成
   - 必要な全モジュールを実装

2. **データベース機能** (`src/db.py`)
   - SQLiteベースのデータベース管理
   - 4つのテーブル実装:
     - `market_prices`: JEPX市場価格
     - `area_generation`: エリア発電実績
     - `pv_profiles`: 太陽光発電プロファイル
     - `simulation_results`: シミュレーション結果
   - CRUD操作の実装

3. **データ正規化** (`src/normalize.py`)
   - JEPXデータのパース
   - エリア発電実績のパース
   - kW→kWh変換
   - 欠損データの補完

4. **PVプロファイル処理** (`src/pv_profile.py`)
   - 1時間データから30分データへの変換
   - エリア太陽光実績による補正機能
   - CSVバリデーション機能

5. **蓄電池シミュレーション** (`src/battery.py`)
   - `BatterySimulator`クラス実装
   - 充放電ロジック
   - SOC管理
   - 2つの放電方式:
     - 上位価格コマ方式
     - 価格しきい値方式

6. **売上計算** (`src/revenue.py`)
   - 蓄電池あり/なしの売上計算
   - サマリー計算（日別、月別、時間帯別）
   - 比較機能

7. **可視化** (`src/visualization.py`)
   - Plotlyベースのグラフ
   - 発電量と価格の可視化
   - 蓄電池挙動の可視化
   - 売上分析グラフ

8. **Streamlitアプリ** (`app.py`)
   - 5つのページ:
     1. ホーム
     2. データ取得
     3. 発電所登録
     4. シミュレーション実行
     5. 結果分析
   - インタラクティブなUI
   - CSVアップロード機能
   - 結果のダウンロード機能

### MVP版の制限事項

以下の機能は未実装（CSV手動アップロードで代替）:

1. JEPX公式サイトからの自動データ取得
2. 各電力会社サイトからの自動データ取得
3. エリア太陽光実績による詳細な補正
4. 託送料、手数料、インバランス、消費税の考慮

## 技術選択の理由

### Streamlit
- 迅速なプロトタイピングが可能
- データサイエンス向けのUIコンポーネントが豊富
- Pythonのみで完結

### SQLite
- 軽量でセットアップ不要
- ローカルPC上での動作に最適
- SQLAlchemyによる拡張性

### Plotly
- インタラクティブなグラフ
- Streamlitとの統合が容易
- ズーム、パンなどの操作が可能

### データの準備

1. **市場価格データ** (`docs/sample_data.md`参照)
   - フォーマット: `datetime,area,area_price_yen_per_kwh`
   - 30分単位

2. **太陽光発電量データ**
   - フォーマット: `datetime,generation_kwh`
   - 1時間単位（自動で30分に変換）

### シミュレーション手順

1. データ取得ページで市場価格をアップロード
2. 発電所登録ページで発電所情報と発電量をアップロード
3. シミュレーション実行ページで条件を設定して実行
4. 結果分析ページで結果を確認・ダウンロード

## 蓄電池シミュレーションの詳細

### 基本ロジック

1. **日単位での最適化**
   - 各日のJEPX価格から放電タイミングを決定
   - 高価格時間帯に最大放電

2. **充電ロジック**
   - 太陽光発電から充電
   - 売電上限超過分を優先的に充電
   - 低価格時間帯にも充電（将来の高価格に備える）

3. **放電ロジック**
   - 放電候補スロットで最大放電
   - SOC、最大放電出力、PCS容量、売電上限を考慮

### パラメータ

- `battery_capacity_kwh`: 蓄電池容量
- `max_charge_kw`: 最大充電出力
- `max_discharge_kw`: 最大放電出力
- `charge_efficiency`: 充電効率（通常0.95）
- `discharge_efficiency`: 放電効率（通常0.95）
- `initial_soc_percent`: 初期SOC
- `min_soc_percent`: 最小SOC（通常10%）
- `max_soc_percent`: 最大SOC（通常90%）

## デバッグ情報

### よくあるエラー

1. **"データが見つかりません"**
   - 原因: データベースに必要なデータが登録されていない
   - 解決: データ取得ページで対象期間のデータをアップロード

2. **"datetime列のパースに失敗"**
   - 原因: CSVのdatetime列が不正なフォーマット
   - 解決: `YYYY-MM-DD HH:MM`形式に修正

3. **蓄電池の動作が期待と異なる**
   - 原因: SOC範囲や充放電出力の設定が不適切
   - 解決: パラメータを見直す
